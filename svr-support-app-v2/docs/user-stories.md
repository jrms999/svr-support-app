# User Stories Backlog (SVR Support App)

## Resident
- As a resident, I want to log in securely so my information is protected.
- As a resident, I want to see my utility balance/status so I know where I stand.
- As a resident, I want to pay utilities easily (mobile-first) to avoid arrears.
- As a resident, I want clear health and crisis contacts in one tap when I need help.
- As a resident, I want local support info based on my SVR site.
- As a resident, I want to read SVR notices so I donâ€™t miss important updates.

## Staff (Phase 2)
- As staff, I want to post notices so residents are informed quickly.
- As staff, I want to update utility balances securely and transparently.
- As staff, I want to manage location-specific resources without editing code.

## Non-functional / safeguarding
- As a user, I want large buttons and clear layouts so the app is easy to use.
- As SVR, we want no sensitive medical data stored in the app.
- As SVR, we want least-privilege access and an audit trail for staff changes (phase 2).
